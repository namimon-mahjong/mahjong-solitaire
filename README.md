# 麻雀ソリティア

麻雀牌で遊ぶソリティア型パズルゲームのPoCです。
山に積まれた牌をめくりながら面子候補を作り、最終的に4面子1雀頭を完成させることを目指します。

## 技術構成

- React 19
- TypeScript
- Vite
- フロントエンドのみで完結

## デプロイ

GitHub Actionsでビルドし、GitHub Pagesへ公開します。

## 牌

以下の麻雀牌を各4枚ずつ使用します。

- 数牌: 萬子、筒子、索子の1から9
- 字牌: 東、南、西、北、白、發、中

牌姿には`riichi-mahjong-tiles`のSVG素材を使用します。

## 画面

- 山エリア
  - 14の山を表示します。
  - 各山の一番上の牌だけが操作できます。
  - 面子候補は山の上に重なったセットとして残ります。
- ステージ
  - 完成した面子と雀頭を置きます。
  - 4面子1雀頭が揃うとクリアです。
- クリア画面
  - クリア後は山を非表示にします。
  - CLEAR演出、使用牌、役判定結果を表示します。

## 基本操作

山の一番上にある牌をドラッグして、別の山の一番上の牌へ重ねます。

2枚が面子候補として有効な場合、その2枚は重ね先の山の上にセットとして残ります。
このとき、ドラッグ元の山だけが1枚めくられます。
重ね先の山の下の牌は、候補が完成または雀頭として送られるまでめくられません。

## 面子候補

完成していない2枚でも、将来面子になり得る場合は候補として重ねられます。
面子候補の牌は昇順に並びます。

### 数牌の候補

同じ種類の数牌で、差が1または2の2枚は順子候補になります。

例:

- 1萬 + 2萬
- 1萬 + 3萬
- 7索 + 8索

同じ牌2枚は刻子候補になります。

例:

- 5筒 + 5筒

### 字牌の候補

字牌は同じ牌2枚だけが刻子候補になります。

例:

- 東 + 東
- 白 + 白

東 + 南のような異なる字牌同士は候補にできません。

## 面子の完成

面子候補に山の一番上の牌を重ねて、3枚の面子を完成させます。
候補側をドラッグして山の牌へ重ねることもできます。

完成した面子はステージへ送られます。
ステージ上の面子は昇順に並びます。

### 数牌の面子

以下のどちらかで完成します。

- 順子: 123、234、...、789
- 刻子: 111、555など同じ牌3枚

891や912のような循環する組み合わせは不可です。

### 字牌の面子

同じ字牌3枚でのみ完成します。

例:

- 東東東
- 白白白
- 中中中

東南西のような組み合わせは不可です。

## 雀頭

同じ牌2枚の刻子候補は、いつでも雀頭としてステージへ送れます。

送る方法:

- 候補セットをステージへドラッグする
- 候補セットに表示される「雀頭へ」ボタンを押す

雀頭は1組だけ作れます。

## クリア条件

以下が揃うとクリアです。

- 面子4組
- 雀頭1組

最後に使った牌を自摸牌として扱います。
クリア後は`riichi-ts`を使って簡易的な役判定を行います。

## 開発

```bash
npm install
npm run dev
```

```bash
npm run build
```
